Common Questions

What's the difference between a website and a web app?

A website presents information. A web application does things — it processes user input, stores data, executes logic, integrates with external services, and returns personalized output. If users log in, make purchases, manage accounts, or interact with dynamic content, you need a web application.

Do I own the code?

Yes. All custom code we write for you is yours. Upon final payment, you receive full ownership of the source code. We can host it for you or help you deploy it on your own infrastructure — your choice. We never retain ownership or control of anything we build for a client.

What technology do you build with?

Our stack depends on project requirements. We work primarily with modern JavaScript/TypeScript, Node.js, and various database and cloud platforms. We choose technology based on what's best for your specific project, not what's trendy. We'll explain our recommendations in plain English during discovery.

Can you work on an existing app someone else built?

Possibly — it depends on the codebase quality and technology stack. We'll review what you have before committing to any work. If the existing code is too messy to safely build on, we'll tell you honestly rather than taking money to dig into a pile we can't fix.
